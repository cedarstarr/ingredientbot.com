## Phase 3 — Three Options

### Option A — Task-Oriented ("What are you doing right now?")
Groups by user intent: Cook → Store → Plan → Browse. Import becomes a Kitchen action (you're importing to cook). Insights folds into Account.

```
Kitchen
  Cook (/kitchen)
  Import Recipe (/import)
  Pantry (/pantry)
  Meal Plan (/meal-plan)

My Recipes
  Saved (/saved)
  Collections (/collections)
  History (/history)

Account
  Usage & Insights (/dashboard)
  Upgrade (/upgrade)
  Settings (/settings)
```

**UX rationale**: Mirrors the natural cook-session flow. Putting Import in Kitchen reinforces "I imported this to use it now." Collapsing Insights into Account reduces visual noise in the sidebar.
**SEO rationale**: No URL changes — sidebar only.
**Tradeoffs**: Insights buried under Account reduces its visibility as a premium conversion surface. Import under Kitchen is logically defensible but slightly surprising.

---

### Option B — Noun-Led Feature Domains (CURRENT)
Each section is a named domain of the app. Kitchen owns active cooking operations. Recipes owns saved content. Insights stands alone as a usage dashboard. Account owns user identity and billing.

```
Kitchen
  Cook (/kitchen)
  Pantry (/pantry)          [expiry badge]
  Meal Plan (/meal-plan)

Recipes
  Saved (/saved)
  Collections (/collections)
  Recently Viewed (/history)
  Import (/import)

Insights
  Insights (/dashboard)

Account
  Upgrade (/upgrade)
  Settings (/settings)
```

**UX rationale**: Clear, scannable noun labels. Each section is a domain, not a task. Import under Recipes is correct because importing *produces* a recipe. Insights as a top-level section keeps usage stats visible to nudge free→premium conversion.
**SEO rationale**: No impact — authenticated sidebar.
**Tradeoffs**: Four sections creates more visual structure than strictly necessary for 9 links; could feel over-categorized to new users.

---

### Option C — Progressive Disclosure (3-section minimal)
Collapses to 3 top-level sections: Kitchen (active tools), Library (all recipe content), You (identity and billing). Fewer sections, broader labels.

```
Kitchen
  Cook (/kitchen)
  Pantry (/pantry)          [expiry badge]
  Meal Plan (/meal-plan)

Library
  Saved (/saved)
  Collections (/collections)
  History (/history)
  Import (/import)

You
  Insights (/dashboard)
  Upgrade (/upgrade)
  Settings (/settings)
```

**UX rationale**: Fewer cognitive sections. "Library" is a stronger content metaphor than "Recipes" (recipes are also ingredients for meals, which creates mild ambiguity). "You" is a common mobile IA shorthand for account.
**SEO rationale**: No impact.
**Tradeoffs**: "Library" and "You" are vaguer labels. "You" in particular may confuse new users who expect "Account" or "Profile." Insights is now one of three Account-level items and loses prominence.
