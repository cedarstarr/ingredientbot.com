## Systemic Fix 1 — `.env` NEXTAUTH_URL

Local `.env` had been overwritten (by an earlier `vercel env pull` or similar) with:

```
NEXTAUTH_URL="https://ingredientbot.com"
```

This caused NextAuth to redirect the local dev server's sign-in flow to production, where the staging test user doesn't exist. Changed to:

```
NEXTAUTH_URL="http://localhost:3010"
```

`.env` is gitignored so this is a local-only fix; no commit.

## Systemic Fix 2 — Seed staging test user

The staging Postgres at `ballast.proxy.rlwy.net:38595/railway` was missing the test user entirely. Reseeded:

```bash
npx tsx scripts/seed-test-user.ts
# → TEST_USER: test@test.com bcryptMatch=true
```
