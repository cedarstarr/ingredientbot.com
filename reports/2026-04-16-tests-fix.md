## Root Causes & Key Findings

### 1. padhr.com — 850 failures were one bug
Signup route was throwing Prisma `P2011` (null constraint on `backupCodes` array) on every request. Agent added `backupCodes: []` to the `user.create()` payload. Sampled 54 previously-failing tests after fix: 53 pass.

The 8-hour runtime was Turbopack dev-server cold-compile stalls under serial (`workers: 1`) test load, not genuine failures — test timeouts fired before routes JIT-compiled.

### 2. ingredientbot.com — missing DB enum migration
Schema declared `ChefPersonality` + `MealType` enums but columns were TEXT in DB; writes 500'd with `type "public.chef_personality" does not exist`. Agent added migration + applied to staging. **⚠️ Production migration needed** (`DATABASE_URL=<prod> npx prisma migrate deploy`) — build script doesn't run `migrate deploy`.

### 3. shkdwn.com — mobile auth cookie propagation
~70 of 124 failures are iPhone 14 WebKit profile failing to propagate NextAuth cookie after `router.push()`. Prior-session work moved to `window.location.href`; still need to validate under WebKit and possibly adjust middleware.

### 4. matchmymajor.ai — `ai@6` SDK cascade
`TypeError: controller[kState].transformAlgorithm is not a function` in WebServer stderr caused a cascade of mobile timeouts. Most "240 failures" are cascade from this Node stream bug, not 240 distinct bugs. Real bug remaining: `/college-search` client React error.

### 5. Cross-cutting test-code smells
- **Regex false positives for `/500/`** — many test files guard against "Internal Server Error" with `/500|internal server error/i`, but "500" matches legitimate content like "500+ Top Employers" and RSC stream timestamps. Fixed in shkdwn, padjobs, dailypriceguru, newsbro — should audit remaining repos.
- **Streaming loading.tsx race** — tests that `waitForURL()` then immediately `textContent('body')` snap the shell before content hydrates. Fixed in gurumind by requiring an `expect(...).toBeVisible()` before reading. Pattern likely repeats elsewhere.
- **Mobile viewport visibility** — Tailwind `sm:` breakpoint (640px) hides elements on iPhone 14 (390px); several repos need mobile audit of hidden CTAs.

### 6. Environmental issues (blockers, not code)
- **padhr.com / padjobs.com** may need fresh Railway staging DBs (migrated from hrmoz/defconjobs — old DBs don't apply).
- **newsbro.com** test user login appears to not be authenticating `test@test.com` — re-seed staging DB via `scripts/seed-test-user.ts`.
- **dailypriceguru.com** mobile login timeouts on Pixel 5 viewport — likely test user `emailVerified` gap or hydration timing.
