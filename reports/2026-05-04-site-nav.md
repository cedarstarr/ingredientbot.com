## Phase 3 — Three Menu Options

### Option A — Organize by User Task/Goal

Group by cooking journey stage.

```
COOK
  Kitchen
  Pantry  [badge]
  Meal Plan

DISCOVER
  Saved Recipes
  Collections
  Import Recipe
  History

ACCOUNT
  Insights
  Upgrade
  Settings
```

- UX: Journey-aligned (cook → discover → account). Collections and Import move out of ACCOUNT to where they semantically belong.
- Tradeoff: "Discover" is slightly abstract; History feels miscategorized here but doesn't fit "Cook" either.

---

### Option B — Flat List with Visual Separators

Frequency-ordered flat list, separators replace section labels.

```
Kitchen
Pantry            [badge]
Saved Recipes
─────────────────
Collections
History
Meal Plan
Import Recipe
─────────────────
Insights
Upgrade
Settings
```

- UX: Minimal cognitive load — no labels, pure frequency order. Power-user friendly.
- Tradeoff: Less discoverable for new users; no hierarchy signals.

---

### Option C — Refined 3-Section with Correct Groupings (Recommended)

Keep 3 sections but fix semantic groupings. Pull Collections and Import into a RECIPES section.

```
KITCHEN
  Kitchen
  Pantry  [badge]
  Meal Plan

RECIPES
  Saved Recipes
  Collections
  History
  Import Recipe

ACCOUNT
  Insights
  Upgrade
  Settings
```

- UX: Corrects the semantic mismatch. KITCHEN = active cooking. RECIPES = recipe library. ACCOUNT = personal account/settings. Same 3-section pattern as current nav, just re-sliced.
- Tradeoff: Minimal disruption to existing structure. Upgrade remains in ACCOUNT, which is acceptable — it's an account action.

---

## Phase 4 — Recommendation

**Option C applied.**

Rationale: The current nav structure is sound (3 sections) but has a semantic flaw — Collections and Import Recipe appear under ACCOUNT when they belong with recipe management. Option C fixes that with the smallest possible change: same visual pattern, same section count, just correct groupings. LayoutDashboard unused import also removed.
