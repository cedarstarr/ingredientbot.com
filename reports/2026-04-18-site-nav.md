## Phase 3 — Three Menu Options

### Option A — Organize by User Task/Goal

Group by cooking journey stage.

```
COOK
  Kitchen
  Import Recipe
  Pantry  [badge]

RECIPES
  Saved Recipes
  History
  Collections

PLAN
  Meal Plan

ACCOUNT
  Dashboard
  Upgrade
  Settings
```

- UX: Mirrors mental flow — cook → manage → plan
- SEO: No impact (auth-gated)
- Tradeoffs: Section headers add visual weight; power users who jump between tools may find grouping unnecessary friction

---

### Option B — Flat with Dividers (Content Type)

Reorganized flat list with subtle visual dividers.

```
Kitchen
Pantry            [badge]
Import Recipe
─────────────
Saved Recipes
Collections
History
─────────────
Meal Plan
Dashboard
─────────────
Upgrade
Settings
```

- UX: Familiar flat pattern, lowest cognitive load, no section labels
- SEO: No impact
- Tradeoffs: Dividers require a small UI addition; Upgrade as a permanent top-level item may feel pushy to paid users

---

### Option C — Kitchen-First + Footer Zone (Recommended)

Frequency-ordered primary nav; low-frequency account tools move to footer.

```
PRIMARY NAV (top, scrollable)
  Kitchen         ← daily use, always first
  Pantry          [badge]
  Saved Recipes
  Collections
  History
  Import Recipe
  Meal Plan

FOOTER NAV (bottom zone, above sign-out)
  Dashboard
  Upgrade         ← resolves orphan
  Settings
```

- UX: Kitchen is the core value prop — it should always be first. Dashboard/Settings are infrequent and belong with sign-out controls. /upgrade is accessible without being visually dominant.
- SEO: No impact
- Tradeoffs: Slightly more UI structure in the footer zone; Upgrade shows for all users including paid (no tier check needed — keeps component simple)

---

## Phase 4 — Recommendation

**Option C applied.**

Rationale: The kitchen is the product's heart — frequency ordering puts daily-use tools where users expect them. Demoting Dashboard, Upgrade, and Settings to a footer zone reduces visual noise in the primary nav without hiding those routes. Adding /upgrade to the footer nav resolves the only orphaned page without an intrusive upsell in the main tool list.
