## Phase 2 — Findings

- **Coverage**: 10/10 non-auth, non-admin app routes reachable from primary sidebar = **100%**
- **Orphaned**: none
- **Dead nav links**: none
- **Issues identified despite 100% coverage:**
  1. `/dashboard` ("Insights") sits under Account next to Settings/Upgrade — semantic mismatch (usage stats are not account settings).
  2. `/import` placed under Kitchen — functionally it's a content-acquisition step (creates a recipe), not a cooking action.
  3. Privacy/Terms not reachable from inside the app — only from public landing/legal pages.
  4. Sidebar group label "Library" is generic; "Recipes" is more specific to the product.

## Phase 3 — Three Options

### Option A — Verb-led (by user task)
- COOK NOW: /kitchen, /import
- PLAN: /meal-plan, /pantry
- MY RECIPES: /saved, /collections, /history
- ACCOUNT: /dashboard, /upgrade, /settings

### Option B — Noun-led (by content type/feature)
- KITCHEN: /kitchen (Cook), /pantry, /meal-plan
- RECIPES: /saved, /collections, /history (Recently Viewed), /import
- INSIGHTS: /dashboard
- ACCOUNT: /upgrade, /settings
- (Footer in sidebar) Privacy · Terms

### Option C — Hybrid (kitchen-first, action-promoted)
- Top CTA button: Cook (/kitchen)
- KITCHEN TOOLS: /pantry, /import, /meal-plan
- LIBRARY: /saved, /collections, /history
- Inline in user row: /dashboard, /upgrade, /settings (small)
- (Footer) Privacy · Terms

## Phase 4 — Recommendation

**Option B** — cleanest information scent. Promoting `/dashboard` out of "Account" fixes a real misclassification (usage stats are not settings). Moving `/import` into Recipes is honest — it produces a recipe. "Recently Viewed" is more specific than "History." Option A's "Cook Now" misleadingly groups Kitchen with Import; Option C reorganizes too much chrome for the size of the IA problem.

**SEO note:** All app routes are auth-gated (non-indexable), so nav changes have no organic-search impact. Public surface is `/`, `/privacy`, `/terms`, `/r/[slug]`. Internal-linking SEO is unaffected.

## Phase 5 — Applied

Applied **Option B** to `src/components/app-nav.tsx`:

- Renamed `/kitchen` label "Kitchen" → "Cook"
- Removed `/import` from KITCHEN_LINKS; added it (as last item) to RECIPES_LINKS
- Renamed `/history` label "History" → "Recently Viewed"
- Renamed `/saved` label "Saved Recipes" → "Saved"
- Section heading "Library" → "Recipes"
- Promoted `/dashboard` (Insights) into its own new INSIGHTS section
- ACCOUNT_LINKS reduced to `/upgrade`, `/settings`
- Added Privacy · Terms link row at bottom of sidebar nav (above the user row), inside the existing scroll container, with `dark:` color tokens via `text-muted-foreground/70` and `hover:text-foreground` (CSS variable tokens, no inline style, focus-visible ring on each link)

**Build:** `npm run build` clean.
