## Manual Review Items

These were audited and intentionally left unchanged:

1. **`src/app/global-error.tsx`** — `bg-[#0f0f0f]`, `text-[#f5f5f5]`, `text-[#a3a3a3]`
   Renders as a standalone `<html>/<body>` replacement outside Next.js's layout and ThemeProvider. CSS vars from `:root` would technically work, but this is a forced-dark error page — replacing with `bg-background` would show a light background in light-mode sessions. Intentional.

2. **`src/app/(app)/kitchen/cook/layout.tsx`** — `bg-[oklch(0.14_0.01_50)]`
   Forced-dark background for the full-screen immersive cook mode. Intentional — cook mode is always dark regardless of user theme preference.

3. **`src/components/kitchen/cooking-mode-client.tsx`** — `bg-white/15`, `border-white/10`, `bg-white/4`, `text-white`, `text-white/*`
   Alpha-composited overlays on the forced-dark cook-mode surface. Not raw palette violations — these are deliberate translucent layering on a known-dark background.

4. **`src/emails/*.tsx`** — `style={{ color: '#...', background: '#...' }}`
   React Email inline styles. Email clients cannot use CSS custom properties; hardcoded values are required. Out of scope per audit instructions.

5. **`src/components/recipe/print-recipe-view.tsx`** — `print:border-gray-800`, `print:text-black`, `print:text-gray-*`, etc.
   All under `print:` variant. Print stylesheets need concrete color values; CSS vars do not reliably render in print mode. Intentional.

6. **`src/components/recipe/collection-detail-client.tsx`, `collections-client.tsx`, `collection-picker.tsx`, `history-client.tsx`** — `style={{ color: collection.color }}`, `style={{ backgroundColor: c }}`
   Dynamic user-chosen collection colors — legitimate dynamic numeric/value context per CLAUDE.md.
