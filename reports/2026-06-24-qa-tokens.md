## Exempt Files (violations are intentional)

### `src/app/opengraph-image.tsx`
All `rgba(...)` values are inside a Satori JSX renderer. Satori cannot process CSS custom properties — hardcoded colors are required here.

### `src/emails/*.tsx` (password-reset, verify-email, welcome)
React Email components require inline styles — CSS classes are not supported in email clients. Hex values (`#e57c2c`, `#111`, `#555`, `#999`) match the site's orange brand color and are correct for the email context.

### `src/components/recipe/print-recipe-view.tsx`
The file has an explicit file-level comment (`// design-system-exempt: print-only grayscale ink values are intentional`) documenting that `print:text-gray-N`, `print:border-gray-N`, `print:bg-gray-N` classes are required for print stylesheets, which cannot use HSL CSS variables.

### `src/components/kitchen/cooking-mode-client.tsx`
`bg-white/N` and `text-white/N` classes are opacity-modified Tailwind alpha variants used intentionally on a full-screen dark cooking overlay. The `style={{ gridTemplateColumns: ... }}` is a dynamic numeric value (allowed per CLAUDE.md). No semantic tokens exist for these — these are correct as-is.

### Collection `style={{ backgroundColor: ... }}` / `style={{ color: ... }}`
In `collection-detail-client.tsx`, `collection-picker.tsx`, `collections-client.tsx`, `history-client.tsx` — all use `collection.color` which is a user-defined runtime value from the DB. These are legitimate dynamic inline styles.
